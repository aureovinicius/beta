3. Sistemas operacionais e servidores

Virtualização e serviços de diretório

A virtualização permite que um único hardware físico hospede múltiplas máquinas virtuais (VMs) isoladas, cada uma com seu próprio sistema operacional, por meio de um hipervisor que abstrai e particiona os recursos físicos (CPU, memória, disco, rede). É a base da consolidação de servidores, da nuvem e da hiperconvergência, aumentando a utilização de hardware e a flexibilidade operacional.

Os serviços de diretório, por sua vez, centralizam informações sobre usuários, computadores e recursos de rede, fornecendo autenticação e autorização unificadas. O Active Directory (AD) da Microsoft e o protocolo LDAP são os pilares dessa área, essenciais para gestão de identidades, single sign-on e aplicação de políticas em ambientes corporativos.

Hipervisores tipo 1 e tipo 2

O hipervisor tipo 1 (bare-metal) roda diretamente sobre o hardware, sem SO hospedeiro, oferecendo melhor desempenho e isolamento — exemplos: VMware ESXi, Microsoft Hyper-V, KVM (integrado ao kernel Linux) e Xen. O hipervisor tipo 2 (hosted) roda sobre um sistema operacional existente — exemplos: VMware Workstation, Oracle VirtualBox — sendo adequado a desktops e laboratórios, não a produção.

O KVM transforma o kernel Linux em hipervisor tipo 1, frequentemente combinado ao QEMU (emulação) e gerenciado por libvirt. O VMware vSphere é a plataforma corporativa cujo hipervisor é o ESXi e cujo gerenciamento central é o vCenter, que habilita recursos como vMotion (migração ao vivo de VMs), DRS (balanceamento de recursos) e HA (reinício automático de VMs em caso de falha de host).

Virtualização x contêineres

Enquanto a virtualização tradicional virtualiza o hardware (cada VM tem um SO completo), os contêineres (Docker, containerd) virtualizam o sistema operacional: compartilham o mesmo kernel do host e isolam apenas o espaço de usuário (via namespaces e cgroups). Contêineres são mais leves, iniciam em segundos e têm menor overhead, mas oferecem isolamento menos rígido que as VMs.

Não se deve confundir os dois: VMs trazem isolamento forte e SO próprio (melhor para multitenancy e cargas heterogêneas); contêineres trazem densidade e portabilidade (melhor para microsserviços). Orquestradores como Kubernetes gerenciam contêineres em escala. É comum a banca testar essa distinção: contêiner não inclui um SO completo nem hipervisor.

Active Directory

O Active Directory Domain Services (AD DS) é o serviço de diretório da Microsoft que organiza objetos (usuários, grupos, computadores) em uma estrutura hierárquica: floresta (forest) contém domínios; domínios contêm unidades organizacionais (OUs). Os controladores de domínio (DCs) autenticam usuários, tipicamente via Kerberos (autenticação por tickets), com NTLM como fallback legado.

O AD integra DNS (essencial para localizar serviços via registros SRV), replicação multi-master entre DCs e o catálogo global (GC) para buscas em toda a floresta. Confiança (trust) entre domínios/florestas permite acesso entre eles. O AD é a base do single sign-on e da aplicação de GPOs no ambiente Windows corporativo.

LDAP e diretórios

O LDAP (Lightweight Directory Access Protocol — RFC 4511) é o protocolo padrão de acesso a serviços de diretório, operando tipicamente na porta 389 (e 636 para LDAPS, sobre TLS). Organiza dados em uma árvore hierárquica (DIT) com entradas identificadas por DN (Distinguished Name), compostas de atributos (cn, ou, dc). O AD é compatível com LDAP, mas vai além ao agregar Kerberos, GPOs e replicação própria.

Implementações abertas incluem OpenLDAP. O LDAP é otimizado para leituras frequentes e poucas escritas (consultas de autenticação/autorização), diferindo de um banco relacional transacional. Bind LDAP é a operação de autenticação; pesquisas usam filtros. Distinguir o protocolo LDAP (a forma de acesso) do serviço de diretório (o repositório, como AD ou OpenLDAP) é importante.

💡 Exemplo aplicado

Item comentado: 'Os contêineres, por executarem um sistema operacional convidado completo sobre um hipervisor tipo 1, fornecem o mesmo nível de isolamento de hardware das máquinas virtuais.' — ERRADO. Contêineres NÃO executam um SO convidado completo nem dependem de hipervisor: compartilham o kernel do host e isolam apenas o espaço de usuário via namespaces e cgroups. Por isso são mais leves, porém com isolamento menos rígido que VMs.

Pontos-chave

  • Hipervisor tipo 1 (bare-metal: ESXi, Hyper-V, KVM, Xen) roda sobre o hardware; tipo 2 (hosted: VirtualBox) sobre um SO
  • KVM torna o kernel Linux um hipervisor tipo 1; vSphere usa ESXi + vCenter (vMotion, DRS, HA)
  • VMs virtualizam hardware (SO completo); contêineres virtualizam o SO (compartilham kernel)
  • AD organiza floresta > domínio > OU; autentica via Kerberos (NTLM legado) e depende de DNS
  • LDAP é o protocolo de acesso a diretórios (porta 389/636); AD é compatível com LDAP
  • DN identifica entradas no DIT; LDAP é otimizado para leitura
  • AD adiciona Kerberos, GPOs, replicação multi-master e catálogo global ao diretório
