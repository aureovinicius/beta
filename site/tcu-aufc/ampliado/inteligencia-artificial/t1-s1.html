2. Redes neurais e deep learning

Treinamento e aplicações

Treinar uma rede neural significa ajustar seus pesos para minimizar uma função de perda (loss), que mede o erro entre as previsões e os valores esperados. O algoritmo central é o backpropagation (retropropagação do erro), que calcula os gradientes da perda em relação a cada peso usando a regra da cadeia, propagando o erro da saída para as camadas iniciais. Esses gradientes alimentam um otimizador (gradiente descendente e variantes) que atualiza os pesos iterativamente.

O grande desafio do treinamento é a generalização: o modelo deve aprender padrões úteis e não decorar os dados de treino. O equilíbrio entre underfitting (modelo simples demais) e overfitting (modelo que decora ruído) é gerenciado por técnicas de regularização e por uma metodologia rigorosa de divisão de dados em treino, validação e teste.

Backpropagation e gradiente descendente

O backpropagation aplica a regra da cadeia do cálculo para obter, de forma eficiente, o gradiente da função de perda em relação a todos os pesos da rede. O otimizador então atualiza os pesos no sentido oposto ao gradiente (gradiente descendente), reduzindo o erro.

Variantes importantes: o gradiente descendente estocástico (SGD) usa um exemplo por vez; o mini-batch processa pequenos lotes (o mais comum). A taxa de aprendizado (learning rate) controla o tamanho do passo: alta demais diverge, baixa demais converge lentamente. Otimizadores adaptativos como Adam, RMSProp e Adagrad ajustam a taxa por parâmetro e aceleram a convergência. Uma época (epoch) corresponde a uma passagem completa por todo o conjunto de treino.

Overfitting, underfitting e o dilema viés-variância

O underfitting ocorre quando o modelo é simples demais e não captura os padrões dos dados (alto viés/bias). O overfitting ocorre quando o modelo se ajusta demais ao treino, incluindo ruído, e generaliza mal para dados novos (alta variância).

O sintoma clássico de overfitting é uma perda de treino baixa, mas uma perda de validação/teste alta - a divergência entre as duas curvas. O objetivo é o ponto de equilíbrio no trade-off viés-variância, em que o erro de generalização é mínimo.

Técnicas de regularização

Regularização engloba métodos para reduzir o overfitting. A regularização L1 (Lasso) e L2 (Ridge/weight decay) penalizam pesos grandes na função de perda; a L1 também induz esparsidade (zera pesos). O dropout desativa aleatoriamente neurônios durante o treino, forçando redundância e robustez.

Outras técnicas: early stopping (interromper o treino quando a validação para de melhorar), data augmentation (gerar variações dos dados de treino, ex.: rotacionar imagens), batch normalization (estabiliza e acelera o treino) e a obtenção de mais dados. A validação cruzada (k-fold) é uma metodologia robusta para estimar a capacidade de generalização.

Divisão de dados e transfer learning

Para avaliar honestamente um modelo, divide-se os dados em treino (ajuste dos pesos), validação (ajuste de hiperparâmetros e seleção de modelo) e teste (estimativa final, intocada durante o desenvolvimento). Vazamento de dados (data leakage) entre esses conjuntos infla artificialmente as métricas.

O transfer learning reaproveita um modelo pré-treinado em grande volume de dados (ex.: uma CNN treinada na ImageNet) e o ajusta (fine-tuning) para uma tarefa específica com poucos dados. É essencial quando há escassez de dados rotulados, situação comum no setor público.

Aplicações práticas

Deep learning sustenta visão computacional (reconhecimento facial, OCR, detecção de objetos), processamento de linguagem (tradução, sumarização, chatbots), reconhecimento de voz, sistemas de recomendação, detecção de fraudes e anomalias e manutenção preditiva.

No controle externo, há aplicações em mineração de dados de contratos para detectar indícios de fraude, classificação automática de processos, análise de notas fiscais e cruzamento de bases para identificar irregularidades.

💡 Exemplo aplicado

Durante uma auditoria de um modelo preditivo de risco de irregularidade em licitações, o auditor observa acurácia de 99% no treino, mas apenas 62% em dados reais novos. Trata-se de um caso clássico de overfitting: o modelo decorou o conjunto de treino. Recomendações: aplicar regularização (dropout, L2), early stopping, mais dados e validação cruzada. Item Cebraspe: 'O dropout é uma técnica de regularização que, durante o treinamento, desativa aleatoriamente parte dos neurônios, reduzindo o overfitting.' Afirmativa CERTA.

Pontos-chave

  • Backpropagation usa a regra da cadeia para calcular gradientes; o otimizador (SGD, Adam) atualiza os pesos.
  • Overfitting = boa no treino, ruim em dados novos (alta variância); underfitting = ruim em ambos (alto viés).
  • Regularização: L1/L2, dropout, early stopping, data augmentation, batch normalization.
  • Dados divididos em treino, validação e teste; o teste só é usado na avaliação final.
  • Taxa de aprendizado controla o tamanho do passo; alta demais diverge, baixa demais é lenta.
  • Transfer learning reaproveita modelos pré-treinados (fine-tuning) quando há poucos dados.
