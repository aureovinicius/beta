2. Malware e ataques a redes

Ataques a redes e aplicações

Os ataques cobrados na prova podem ser organizados por alvo: ataques à DISPONIBILIDADE (DoS/DDoS), ataques baseados em ENGENHARIA SOCIAL (phishing), ataques que exploram FALHAS DESCONHECIDAS (zero-day), ataques de FALSIFICAÇÃO/INTERCEPTAÇÃO em rede (spoofing, DNS poisoning, MITM) e ataques contra APLICAÇÕES (buffer overflow, SQL injection, XSS).

O examinador costuma exigir que você identifique o mecanismo do ataque e a propriedade da segurança violada (confidencialidade, integridade, disponibilidade). Dominar essas correspondências é o que separa o acerto do erro em itens certo/errado.

Ataques à disponibilidade: DoS e DDoS

DoS (Denial of Service): negação de serviço, em que o atacante esgota recursos (banda, CPU, conexões) para tornar o serviço indisponível, partindo geralmente de UMA origem. DDoS (Distributed DoS): negação distribuída, com múltiplas origens coordenadas (uma botnet), o que dificulta o bloqueio e amplifica o volume. Há ataques de amplificação/reflexão (DNS, NTP, memcached) que multiplicam o tráfego. Violam a DISPONIBILIDADE. Defesas: rate limiting, scrubbing centers, CDNs, anti-DDoS de provedores.

Engenharia social: phishing e variantes

Phishing: fraude que induz a vítima a revelar dados sensíveis (senhas, dados bancários) por mensagens falsas que se passam por entidades legítimas. Variantes: spear phishing (direcionado a alvo específico), whaling (alvos executivos), smishing (por SMS), vishing (por voz/telefone) e pharming (redireciona a vítima a site falso, frequentemente via DNS poisoning). Explora o fator humano; a defesa combina conscientização, filtros antispam, DMARC/SPF/DKIM e MFA.

Zero-day e ataques de rede (spoofing, poisoning, MITM)

Zero-day: explora uma vulnerabilidade ainda DESCONHECIDA do fabricante (sem correção/patch disponível) — 'zero dias' de aviso prévio; por isso assinaturas tradicionais não a detectam. IP spoofing: falsifica o endereço IP de origem dos pacotes para se passar por outra máquina ou ocultar a origem. ARP spoofing (ARP poisoning): envia respostas ARP forjadas na LAN para associar o IP da vítima/gateway ao MAC do atacante, permitindo interceptar o tráfego (viabiliza Man-in-the-Middle). DNS poisoning (cache poisoning): corrompe o cache de um resolvedor DNS para que um domínio legítimo resolva para um IP malicioso, redirecionando usuários (base do pharming). MITM intercepta a comunicação entre duas partes que creem falar diretamente.

Ataques a aplicações: overflow, injeção e XSS

Buffer overflow (estouro de buffer): escreve dados além do limite de um buffer na memória, podendo corromper a pilha e executar código arbitrário; explora linguagens sem verificação de limites (C/C++). SQL Injection (SQLi): insere comandos SQL maliciosos em campos de entrada não validados, permitindo ler, alterar ou apagar dados do banco e burlar autenticação (ex.: ' OR '1'='1). XSS (Cross-Site Scripting): injeta scripts maliciosos (JavaScript) em páginas web que serão executados no NAVEGADOR de outros usuários — tipos: stored (armazenado), reflected (refletido) e DOM-based; permite roubar cookies de sessão e sequestrar contas. A defesa central é validação/sanitização de entradas, consultas parametrizadas (prepared statements) para SQLi e output encoding/CSP para XSS.

💡 Exemplo aplicado

Item comentado: 'No ataque de cross-site scripting, o invasor injeta código que é executado no servidor da aplicação web, comprometendo o banco de dados.' — ERRADO. O XSS executa scripts no NAVEGADOR de outros usuários (lado cliente), não no servidor; quem ataca diretamente o banco via comandos injetados é o SQL Injection. Em auditoria, ao avaliar um sistema web do governo, o TCU pode solicitar evidências de testes de segurança (pentest, DAST) cobrindo o OWASP Top 10 e verificar se as entradas são parametrizadas (anti-SQLi) e codificadas na saída (anti-XSS).

🎯 Na prova

A confusão favorita da banca é XSS x SQLi: XSS = cliente/navegador; SQLi = servidor/banco. Outra: dizer que DDoS quebra a confidencialidade — não; ataca a DISPONIBILIDADE. Zero-day é definido pela ausência de correção conhecida, não pela técnica. ARP spoofing é local (camada de enlace, na LAN); IP spoofing falsifica o endereço de rede. Saiba mapear cada ataque à propriedade (CID) que viola.

Pontos-chave

  • DoS parte de uma origem; DDoS é distribuído (botnet); ambos violam disponibilidade.
  • Phishing explora o fator humano; pharming redireciona via DNS poisoning.
  • Zero-day explora falha desconhecida sem patch; escapa de assinaturas.
  • IP spoofing falsifica IP de origem; ARP spoofing viabiliza MITM na LAN.
  • DNS poisoning corrompe o cache para redirecionar a sites maliciosos.
  • SQL Injection ataca o banco (servidor); XSS executa script no navegador (cliente).
  • Buffer overflow estoura limites de memória e pode executar código arbitrário.
  • Defesas de app: validação de entrada, prepared statements (SQLi), CSP/encoding (XSS).
