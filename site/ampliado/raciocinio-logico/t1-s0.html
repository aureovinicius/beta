2. Lógica proposicional

Conectivos e tabela-verdade

A lógica proposicional estuda proposições — sentenças declarativas às quais se pode atribuir um único valor lógico: verdadeiro (V) ou falso (F). Sentenças interrogativas, exclamativas, imperativas e abertas (com variável) não são proposições. A partir de proposições simples, os conectivos formam proposições compostas, cujo valor depende dos valores das partes e do conectivo usado.

A tabela-verdade é o instrumento que define, exaustivamente, o valor de uma proposição composta para cada combinação possível dos valores das simples. Para n proposições simples há 2 elevado a n linhas: 2 proposições geram 4 linhas, 3 geram 8 linhas. Dominar os cinco conectivos é a base de toda a disciplina.

Os cinco conectivos e seus símbolos

Negação (~p ou ¬p): inverte o valor. Conjunção (p ∧ q, "e"): só é V quando ambas são V. Disjunção inclusiva (p ∨ q, "ou"): só é F quando ambas são F. Condicional (p → q, "se p então q"): só é F quando o antecedente é V e o consequente é F. Bicondicional (p ↔ q, "se e somente se"): é V quando os dois lados têm o mesmo valor.

Existe ainda a disjunção exclusiva (p ⊻ q, "ou...ou", mas não ambos): é V quando os valores são diferentes — exatamente o oposto do bicondicional.

Tabela-verdade dos conectivos

Para p e q assumindo VV, VF, FV, FF:

Conjunção (p∧q): V, F, F, F. Disjunção (p∨q): V, V, V, F. Condicional (p→q): V, F, V, V. Bicondicional (p↔q): V, F, F, V. Disjunção exclusiva (p⊻q): F, V, V, F.

Observe que a condicional só falha na segunda linha (V→F = F). Esse é o ponto mais cobrado: "verdadeiro implicando falso é falso; todos os outros casos são verdadeiros".

Ordem de precedência dos conectivos

Quando faltam parênteses, a precedência usual é: ~ (mais forte), depois ∧, depois ∨, depois →, e por último ↔ (mais fraco). Assim, ~p ∧ q → r deve ser lido como ((~p) ∧ q) → r.

Na prática de prova, recomenda-se sempre acrescentar parênteses mentalmente para evitar ambiguidade, pois algumas bancas exigem rigor na leitura da expressão.

Construção passo a passo de uma tabela

Para avaliar (p ∧ q) → ~p: monte colunas para p, q, p∧q, ~p e finalmente (p∧q)→~p. Linha VV: p∧q=V, ~p=F, então V→F=F. Linha VF: p∧q=F, ~p=F, F→F=V. Linha FV: p∧q=F, ~p=V, F→V=V. Linha FF: p∧q=F, ~p=V, F→V=V. Resultado da coluna final: F, V, V, V.

💡 Exemplo aplicado

Questão típica: "Se a proposição 'Goiás arrecada ICMS e o contribuinte sonega' é verdadeira, qual o valor de cada parte?" Como a conjunção só é V quando ambas são V, concluímos que ambas as partes são verdadeiras.

Outro exemplo com condicional: dado que p → q é falsa, descubra p e q. A condicional só é falsa em um caso: antecedente V e consequente F. Logo, necessariamente p é verdadeira e q é falsa. Esse tipo de dedução (extrair valores das partes a partir do valor do todo) é frequentíssimo e mais rápido que montar a tabela inteira.

🎯 Na prova

A pegadinha mais explorada é a condicional: muitos candidatos acham que F→V ou F→F seriam falsos, mas a condicional só é falsa em V→F. Decore: 'só dá errado quando a promessa (antecedente V) é cumprida mas o resultado (consequente) não vem'. Cuidado também para não confundir 'ou' inclusivo (∨) com 'ou...ou' exclusivo (⊻).

Pontos-chave

  • Proposição é sentença declarativa com valor único V ou F.
  • Conjunção (∧): só V se ambas V. Disjunção (∨): só F se ambas F.
  • Condicional (→): só F quando V → F.
  • Bicondicional (↔): V quando os dois lados são iguais.
  • Disjunção exclusiva (⊻): V quando os valores diferem.
  • n proposições simples geram 2^n linhas na tabela-verdade.
  • Precedência: ~, depois ∧, depois ∨, depois →, depois ↔.
