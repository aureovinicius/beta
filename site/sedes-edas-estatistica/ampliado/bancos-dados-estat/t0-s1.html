1. Bancos de dados

SQL

♻ Conteúdo compartilhado com o concurso Auditor-Fiscal da Receita Estadual de Goiás — mesmo assunto cobrado em ambos.

SQL (Structured Query Language) é a linguagem padrão (ANSI/ISO) para definir, manipular e controlar dados em bancos relacionais. É uma linguagem declarativa: o usuário descreve O QUE quer obter, e o SGBD decide COMO obter (via otimizador de consultas). Esse caráter declarativo a diferencia de linguagens procedurais.

SQL é tradicionalmente subdividida em sublinguagens conforme a finalidade dos comandos: DDL (definição), DML (manipulação), DCL (controle de acesso) e DTL/TCL (controle de transações). Algumas bancas incluem ainda a DQL para o SELECT, mas é comum tratar o SELECT dentro da DML — atenção a essa divergência de classificação.

Para a fiscalização, SQL é a ferramenta central: cruzar EFD com NF-e, somar bases de cálculo de ICMS por período, localizar contribuintes omissos e gerar malhas fiscais são, no fundo, consultas SQL.

DDL — Data Definition Language

A DDL define e altera a estrutura (esquema) do banco. Comandos principais: CREATE (cria tabelas, índices, views, esquemas), ALTER (modifica estruturas existentes), DROP (remove objetos) e TRUNCATE (esvazia uma tabela rapidamente).

Pegadinha importante: TRUNCATE é classificado como DDL (não DML), remove todas as linhas sem registrar cada exclusão no log da mesma forma que o DELETE, e em muitos SGBDs provoca commit implícito (não pode ser desfeito por ROLLBACK). DROP remove a própria estrutura; TRUNCATE remove só os dados, mantendo a tabela.

DML — Data Manipulation Language

A DML manipula os dados (linhas) dentro das estruturas: INSERT (insere), UPDATE (altera), DELETE (remove) e SELECT (consulta). O SELECT é o comando mais rico, com cláusulas em ordem lógica: FROM (origem e JOINs), WHERE (filtro de linhas), GROUP BY (agrupamento), HAVING (filtro de grupos), SELECT (projeção de colunas) e ORDER BY (ordenação).

Distinção que cai muito: WHERE filtra linhas ANTES do agrupamento; HAVING filtra grupos DEPOIS do GROUP BY (e pode usar funções de agregação como SUM, COUNT, AVG, MAX, MIN). Funções de agregação ignoram NULL (exceto COUNT(*)).

DCL e TCL — controle de acesso e de transações

A DCL (Data Control Language) controla privilégios: GRANT concede permissões (SELECT, INSERT, etc.) a usuários/papéis; REVOKE as retira. É a base da segurança e da segregação de funções no acesso a dados sigilosos do contribuinte.

A TCL/DTL (Transaction Control Language) gerencia transações: COMMIT confirma e torna permanentes as alterações; ROLLBACK desfaz; SAVEPOINT cria pontos intermediários de retorno. Algumas bancas cobram esses comandos como parte da DCL — verifique o enunciado, mas o mais técnico é separá-los.

JOINs e operações de conjunto

Para combinar tabelas, usam-se JOINs: INNER JOIN (só linhas com correspondência nas duas tabelas), LEFT/RIGHT OUTER JOIN (preservam todas as linhas de um lado, completando com NULL), FULL OUTER JOIN (preserva ambos) e CROSS JOIN (produto cartesiano).

Operadores de conjunto combinam resultados de SELECTs com mesma estrutura: UNION (une eliminando duplicatas), UNION ALL (une mantendo duplicatas — mais rápido), INTERSECT (interseção) e EXCEPT/MINUS (diferença). Pegadinha: UNION elimina duplicatas e ordena/processa mais; UNION ALL não elimina.

💡 Exemplo aplicado

Para identificar contribuintes cujo total declarado de saídas de ICMS em 2025 ultrapassou R$ 1 milhão, mas que não estão no regime normal:

SELECT c.insc_estadual, c.razao_social, SUM(n.valor) AS total FROM nfe n JOIN contribuinte c ON c.insc_estadual = n.insc_emitente WHERE EXTRACT(YEAR FROM n.data) = 2025 GROUP BY c.insc_estadual, c.razao_social HAVING SUM(n.valor) > 1000000 ORDER BY total DESC;

Note o uso de WHERE (filtra notas de 2025), GROUP BY (agrupa por contribuinte), HAVING (filtra somente os grupos cujo total excede 1 milhão) e ORDER BY. Questão típica: 'A cláusula HAVING pode ser usada sem GROUP BY?' — Em geral sim, tratando toda a tabela como um único grupo, mas conceitualmente HAVING filtra grupos e WHERE filtra linhas; trocar os dois é o erro mais cobrado.

🎯 Na prova

Memorize a alocação de cada comando à sua sublinguagem — bancas adoram dizer que TRUNCATE é DML (errado, é DDL) ou que GRANT é DML (errado, é DCL). Outra cilada recorrente: afirmar que HAVING é usada para filtrar linhas (não, ela filtra grupos) ou que DELETE remove a estrutura da tabela (não, só os dados; quem remove a estrutura é DROP).

Pontos-chave

  • SQL é declarativa e padrão ANSI/ISO para bancos relacionais.
  • DDL: CREATE, ALTER, DROP, TRUNCATE (estrutura). DML: INSERT, UPDATE, DELETE, SELECT (dados).
  • DCL: GRANT/REVOKE (acesso). TCL: COMMIT, ROLLBACK, SAVEPOINT (transações).
  • WHERE filtra linhas antes; HAVING filtra grupos após GROUP BY.
  • TRUNCATE é DDL, esvazia rápido e costuma ter commit implícito; DELETE é DML e transacional.
  • Funções de agregação (SUM, AVG, COUNT, MAX, MIN) ignoram NULL, salvo COUNT(*).
  • UNION remove duplicatas; UNION ALL mantém. INNER JOIN só traz correspondências.
